# JSON.stringify() 的 秘密

之前在解决问题的时候发现了一个关于JSON.stringify的奥秘，就拿出来跟大家学习交流一下，

你在开发中一定遇到过这样的场景：

const user = {name:"zhangsan",age: 26}
console.log(user)
//期待：{"name":"zhangsan","age":26}
//结果：[object Object]
1
2
3
4

并没有如想象那样打印出字符串，这是因为从对象到字符串的默认转换是 [object Object]。

当使用JSON.stringify 将对象转换成字符串，然后在控制台中打印，则可获得期待的结果。

console.log(JSON.stringify(user))  //{"name":"zhangsan","age":26}
1

在可视化开发的过程中我就遇到一个问题，拖拽的时候dataTransfer只能传递字符串，如若我们想传递一个dom或者其他对象，就可以采用JSON.stringify将对象字符串序列化。

下面摘抄自MDN的解释：

JSON.stringify(value[, replacer [, space]])，JSON.stringify() 方法将一个 JavaScript 值（对象或者数组）转换为一个 JSON 字符串，如果指定了 replacer 是一个函数，则可以选择性地替换值，或者如果指定了 replacer 是一个数组，则可选择性地仅包含数组指定的属性。

value：将要序列化成 一个 JSON 字符串的值。

replacer (可选)：如果该参数是一个函数，则在序列化过程中，被序列化的值的每个属性都会经过该函数的转换和处理；如果该参数是一个数组，则只有包含在这个数组中的属性名才会被序列化到最终的 JSON 字符串中；如果该参数为 null 或者未提供，则对象所有的属性都会被序列化

space(可选) :指定缩进用的空白字符串，用于美化输出（pretty-print）；如果参数是个数字，它代表有多少的空格；上限为10。该值若小于1，则意味着没有空格；如果该参数为字符串（当字符串长度超过10个字母，取其前10个字母），该字符串将被作为空格；如果该参数没有提供（或者为 null），将没有空格。

光是以上参数，可能就有很多人不知道其存在，继续看看下面几种情况：

  1. 当一个对象很庞大，而我们只需要一个或一些关键信息的时候，如下

    const info = {
	name: 'zhangsan',
	age: 30,
	sex: 'male',
	...
}
console.log(JSON.stringify(info,['name' ]); // {"name" : "zhangsan"}

    firstName: 'zhang',
    lastName: 'san',
    toJSON() {
       return {fullNmae:`${this.firstName}:${this.lastName}`}

}

// {"fullNmae":"zhang:san"}

以上的几种场景说明，JSON.stringify有很强的处理能力，以后可以多多尝试在业务场景中使用，而不再是仅仅做深克隆之类的实现，当然深克隆采用JSON.stringify本就是性能很差的一种实现
